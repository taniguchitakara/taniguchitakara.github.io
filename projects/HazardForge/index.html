Towards Safer Mobile Agents: Scalable Generation and Evaluation of Diverse Scenarios for VLMs

Takara Taniguchi, Kuniaki Saito, Atsushi Hashimoto
OMRON SINIC X
arXiv Code

Abstract

Vision Language Models (VLMs) are increasingly deployed in autonomous vehicles and mobile systems, making it crucial to evaluate their ability to support safer decision-making in complex environments. However, existing benchmarks inadequately cover diverse hazardous situations, especially anomalous scenarios with spatio-temporal dynamics. While image editing models are a promising means to synthesize such hazards, it remains challenging to generate well-formulated scenarios that include moving, intrusive, and distant objects frequently observed in the real world.

To address this gap, we introduce HazardForge, a scalable pipeline that leverages image editing models to generate these scenarios with layout decision algorithms, and validation modules. Using HazardForge, we construct MovSafeBench, a multiple-choice question (MCQ) benchmark comprising 7,254 images and corresponding QA pairs across 13 object categories, covering both normal and anomalous objects. Experiments using MovSafeBench show that VLM performance degrades notably under conditions including anomalous objects, with the largest drop in scenarios requiring nuanced motion understanding.

Overview

Mobile agents encounter two types of scenarios: common scenarios involving ordinary objects in high-risk contexts (e.g., vehicles on the road, pedestrians crossing a lane), and anomalous scenarios involving unusual objects rarely observed in standard datasets (e.g., deer or pigs on the roadway). In both cases, temporal dynamics and spatial relations are crucial for evaluating VLMs' ability to reason about object motion and distance.

MovSafeBench frames the task as direction-oriented question answering: given an image and question, the model selects the safe driving direction — Go Left, Go Straight, or Go Right.

Four Evaluated Scenarios

Motion Scenario

A hazard object is placed at the image center moving toward one side, requiring the agent to understand motion direction and maneuver around it. The object's direction is set opposite to the ground-truth safe direction.

Intrusion Scenario

An object intrudes from outside the field of view into a non-safe lane region, simulating real-world situations where obstacles appear suddenly at the image boundary.

Distance Scenario

A remote object is placed in the safe region while nearby objects appear in the unsafe regions. Tests whether VLMs correctly prioritize nearby hazards over distant ones.

Static Scenario

Nearby objects with a front-facing view are placed in non-safe regions. Serves as a baseline for comparing the effect of spatio-temporal dynamics.

HazardForge Pipeline

HazardForge addresses two core challenges: (i) data scarcity through expansive synthesis of both common and anomalous hazards, and (ii) spatio-temporally diverse generation. The pipeline produces images by combining an image editing model with scenario-specific layout decision algorithms.

1

Layout Decision

The image is horizontally split into three regions corresponding to Go Left, Go Straight, and Go Right. For each scenario type, a conditional mask is computed to determine where the object should be rendered — considering the ground-truth safe direction, object size, and spatial constraints such as vanishing point location for distant objects.

2

Image Editing

An image editing model renders the object into the scene given the input image, conditional mask, and a text prompt specifying category and motion direction (e.g., "Render a raccoon, facing left."). For intrusion scenarios, the image is first outpainted to allow partial objects to appear at the boundary. For distance scenarios, vanishing point detection guides small-mask placement near the horizon.

3

Quality Check

A VLM-based quality check module (Qwen3-VL-30B) independently verifies (i) whether the object is fully inserted and (ii) whether it faces the intended direction. Images that fail either check are discarded, ensuring only well-formed samples enter the benchmark.
